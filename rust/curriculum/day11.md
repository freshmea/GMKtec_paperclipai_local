# Day 11. 에러 처리 (Error Handling)

## 1. 강의 개요
- **일자:** Day 11
- **목표:** Rust의 강력한 에러 처리 메커니즘을 이해하고, `Result`와 `Option`을 활용하여 견고한 코드를 작성하는 법을 익힌다.

---

## 2. 시간대별 세션

### 09:00~10:30: Option 타입: 값이 있을 수도, 없을 수도 있는 상황
- **목표:** `null`의 위험성을 배우고, Rust의 안전한 대안인 `Option<T>`을 이해한다.
- **주요 내용:**
    - **`Option<T>`의 정의:** `Some(T)`와 `None`
    - 왜 `null` 대신 `Option`을 사용하는가? (컴파일 타임의 안전성)
    - `Option` 처리 방법: `match`, `if let`, `unwrap()`, `expect()`
- **실습 지시:**
    - 리스트에서 인덱스로 요소를 찾을 때 `get()` 메서드가 `Option`을 반환하는 과정 관찰하기.
    - `unwrap()`을 사용했을 때 `None`인 경우 발생하는 패닉 확인하기.

### 10:45~12:00: Result 타입: 성공과 실패의 명시적 처리
- **목표:** 함수 실행의 성공과 실패를 타입 시스템으로 다루는 법을 배운다.
- **주요 내용:**
    - **`Result<T, E>`의 정의:** `Ok(T)`와 `Err(E)`
    - 에러를 값으로 취급하기: 함수 반환 타입으로서의 `Result`
    - `match`를 이용한 에러 처리의 정석
- **실습 지시:**
    - 파일 읽기(`std::fs::File::open`)를 시도하고, 성공과 실패(파일 없음) 시나리오를 `match`로 처리하기.

### 13:00~14:30: 에러 전파 (Error Propagation)와 `?` 연산자
- **목표:** 에러를 상위 함수로 깔끔하게 전달하여 코드의 가독성을 높이는 법을 배운다.
- **주요 내용:**
    - **`?` 연산자의 마법:** `Result`를 반환하는 함수 내에서 에러 발생 시 즉시 반환하기.
    - 에러 전파의 흐름: `Err`가 발생하면 함수가 즉시 종료되고 에러가 호출자에게 전달됨.
    - `unwrap()`과 `?`의 결정적 차이점.
- **실습 지시:**
    - 여러 단계의 `Result`를 반환하는 함수들을 작성하고, `?` 연산자를 사용하여 중첩된 `match` 없이 깔끔하게 구현하기.

### 14:45~16:00: 사용자 정의 에러 (Custom Errors)
- **목표:** 프로젝트의 도메인에 맞는 구체적인 에러 타입을 정의한다.
- **주요 내용:**
    - `enum`을 이용한 에러 타입 정의
    - `std::error::Error` 트레이트 구현 (기초)
    - `From` 트레이트를 이용한 에러 타입 변환 (에러 래핑)
- **실습 지시:**
    - `MyError`라는 enum을 만들고, `FileError`와 `ParseError`를 포함시켜 함수에서 반환하기.

### 16:10~18:00: 랩 (Lab) 및 과제
- **목표:** 에러 처리가 포함된 완성도 있는 프로그램을 작성한다.
- **활동:**
    - **안전한 설정 파일 로더(Config Loader) 구현:**
        - 설정 파일(텍스트 파일)을 읽고, 특정 형식(`key=value`)을 파싱하는 프로그램.
        - 파일이 없거나, 형식이 잘못되었거나, 값이 비어있는 경우를 각각의 커스텀 에러로 처리.
    - **과제: 에러 전파 체인 설계**
        - `Read` $\rightarrow$ `Parse` $\rightarrow$ `Validate`로 이어지는 3단계 파이프라인을 구현하고, 각 단계에서 발생하는 에러를 `?`를 통해 최상위 함수까지 전달하기.
