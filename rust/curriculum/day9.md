# Day 9. Struct, Enum, Match

## 1. 강의 개요
- **일자:** Day 9
- **목표:** Rust의 핵심 데이터 모델링 도구인 `struct`와 `enum`을 배우고, 이를 통해 복잡한 도메인을 표현하는 방법을 익힌다.

---

## 2. 시간대별 세션

### 09:00~10:30: Struct 설계와 구현
- **목표:** 데이터를 구조화하고 관련 동작(메서드)을 정의한다.
- **주요 내용:**
    - **Classic Structs:** 필드를 가진 구조체 정의
    - **Associated Functions:** `impl` 블록 내의 `fn Name()` (예: `new()` 생성자 패턴)
    - **Methods:** `self`를 사용하여 인스턴스에 동작을 부여하는 법
- **실습 지시:**
    - `User` 또는 `Rectangle` 구조체를 만들고, 관련 메서드(`area`, `perimeter` 등) 구현하기

### 10:45~12:00: Enum, Option, Match
- **목표:** 상태를 표현하는 강력한 도구인 `enum`과 에러/부재 상황을 다루는 `Option`을 이해한다.
- **주요 내용:**
    - **Enum with Data:** 각 변체가 서로 다른 데이터를 가질 수 있는 Rust만의 강력한 `enum`
    - **Option<T>:** 값이 있을 수도, 없을 수도 있는 상황을 안전하게 처리
    - **Pattern Matching:** `match` 문을 이용한 완벽한 분기 처리
    - **if let:** 특정 패턴만 관심 있을 때 사용하는 간결한 문법
- **실습 지시:**
    - `Message` enum(Quit, Move, Write)을 만들고 `match`로 처리하기
    - `Option` 타입을 사용하여 값이 없는 경우를 안전하게 처리하는 코드 작성하기

### 13:00~14:30: 상태 전이 모델링 실습
- **목표:** `enum`을 사용하여 시스템의 상태(State)를 안전하게 설계한다.
- **주요 내용:**
    - 상태 머신(State Machine)의 개념
    - `enum`의 각 변체에 상태 정보를 담아 불가능한 상태를 컴파일 타임에 차단하기
- **실습 지시:**
    - 주문(Order) 상태(Pending, Paid, Shipped, Delivered)를 `enum`으로 정의하고, 각 상태에서 가능한 동작을 메서드로 구현하기

### 14:45~16:00: 도메인 모델 설계 연습
- **목표:** 현실 세계의 문제를 Rust의 타입 시스템으로 추상화한다.
- **주요 내용:**
    - 요구사항 분석 $\rightarrow$ 데이터 구조 설계 $\rightarrow$ 행위(Method) 정의
    - `struct`와 `enum`의 적절한 조합
- **실습 지시:**
    - 간단한 '도서 관리 시스템'의 데이터 모델 설계 (Book, Author, Library 등)

### 16:10~18:00: 랩 (Lab) 및 과제
- **목표:** 설계한 모델을 코드로 구현한다.
- **활동:**
    - **도서 관리 또는 주문 상태 관리 모델 구현:**
        - 앞서 설계한 모델을 바탕으로 실제 동작하는 코드 작성
        - `enum`을 사용하여 상태 변화가 올바르게 일어나는지 검증
    - **과제: enum 기반 상태 머신 작성**
        - 특정 조건(예: 결제 완료 시에만 배송 가능)에서만 상태가 변할 수 있도록 로직을 엄격하게 제한하는 코드 작성
