# 슬라이드 비주얼 앵커

## 목적

- `slide_images/`에 렌더링된 PDF 첫 장을 기준으로 각 주차의 메시지 톤과 교안 패키지의 연결성을 빠르게 검토한 메모다.
- 범위는 "전 덱 전체 판독"이 아니라 "덱 전면부 시각 검토를 통한 주차별 메시지 정렬 확인"이다.
- 발표자 스크립트와 진행자 노트가 실제 슬라이드의 언어와 어긋나지 않도록 주차별 앵커 문구를 남긴다.

## 주차별 해석 메모

### 1주차

- 슬라이드 표지의 핵심 은유는 "엔진", "궤도", "설계"다.
- 따라서 발표자 스크립트의 1주차 메시지처럼 DX와 AX를 개념 차이로만 풀지 말고, 업무 흐름을 설계 가능한 대상으로 보게 만드는 방향이 맞다.
- 2일차 이후 설명은 "비효율 찾기"에 머무르지 않고 "AI가 달릴 레일을 그린다"는 표현으로 통일하면 슬라이드와의 결이 맞는다.

### 2주차

- 확인한 표지 슬라이드는 1주차의 혼란스러운 진단 영역과 2주차의 구조화된 해결 영역을 대비시킨다.
- 기존 발표자 스크립트의 "프롬프트는 질문이 아니라 업무 지시서"라는 문장은 이 시각 메시지와 잘 맞는다.
- 진행자 노트에서는 2주차 시작 시 1주차 산출물을 반드시 다시 열게 하는 운영 원칙을 더 강하게 지켜야 한다.

### 3주차

- 3주차 표지는 단계형 구조와 파이프라인 이미지를 전면에 둔다.
- 강의 톤도 "자동화 도구 소개"보다 "입력-처리-출력-검수" 흐름을 설계하는 사고 훈련에 초점을 두는 편이 적합하다.
- 진행자 노트의 예외 처리 질문은 이 주차 슬라이드 톤과 잘 맞으므로 유지 가치가 높다.

### 4주차

- 4주차 표지는 텍스트 문서에서 시각화된 결과물로 넘어가는 전환을 매우 직접적으로 보여 준다.
- 발표자 스크립트의 16일차 메시지처럼 "표현을 바꾸는 도구"가 아니라 "업무 전달력을 넓히는 도구"라는 정의가 적절하다.
- 실습 안내는 결과물 미관보다 업무 활용 장면과 설득 목적을 먼저 묻게 구성해야 한다.

## 운영상 권고

- 각 주차 1일차 오프닝에서 해당 주차 대표 표지 슬라이드를 1분 정도 해설하며 "이번 주에 무엇이 바뀌는가"를 먼저 못 박는다.
- 발표자 스크립트의 주차 도입부에는 슬라이드 은유를 그대로 재사용한다. 예: 엔진, 궤도, 설계자, 비주얼 커뮤니케이션.
- anomaly 덱인 `1주차/2-2.pdf`, `1주차/5-1-1.pdf`, `4주차/18-2.pdf`는 본문 스크립트보다 자산 정합성 확인이 우선이다.

## 한계

- 이번 메모는 렌더링된 이미지의 대표 표지 중심 검토다.
- 790페이지 전체를 슬라이드별로 대본화한 문서는 아니며, 전량 대본 작업은 별도 배치가 필요하다.
- 다만 현재 교안 패키지의 주차별 메시지 방향이 실제 슬라이드 전면부와 크게 어긋나지 않는다는 점은 확인했다.
