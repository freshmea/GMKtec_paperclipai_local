# Day 11. 에러 처리 (Error Handling)

## 1. 강의 개요
- **일자:** Day 11
- **목표:** Rust의 에러 처리 모델인 `Result`와 `Option`을 이해하고, 에러를 전파하고 처리하는 견고한 방법을 익힌다.

---

## 2. 시간대별 세션

### 09:00~10:30: Option과 Result - 에러의 기초
- **목표:** 값이 없을 수 있는 상황과 에러가 발생할 수 있는 상황을 타입으로 다루는 법을 배운다.
- **주요 내용:**
    - **Option Type:** `Some(T)`와 `None`을 사용한 값의 부재 표현.
    - **Result Type:** `Ok(T)`와 `Err(E)`를 사용한 성공/실패 표현.
    - **Pattern Matching:** `match`와 `if let`을 사용한 안전한 값 추출.
- **실습 지시:**
    - `divide` 함수를 만들어 0으로 나눌 때 `None` 또는 `Err`를 반환하도록 구현하기.

### 10:45~12:00: 에러 전파와 `?` 연산자
- **목표:** 에러를 호출자에게 효율적으로 전달하는 방법을 배운다.
- **주요 내용:**
    - **The `?` Operator:** 에러 발생 시 즉시 반환하는 문법적 설탕(syntactic sugar).
    - **Error Propagation:** 에러 발생 시 호출자에게 에러를 위로 올리기.
- **실습 지시:**
    - 여러 단계의 함수 호출이 포함된 로직에서 `?`를 사용하여 에러를 전파하는 코드 작성하기.

### 13:00~14:30: Custom Error Types - 나만의 에러 정의하기
- **목표:** 도메인에 맞는 명확한 에러 타입을 설계한다.
- **주요 내용:**
    - **Enums for Errors:** 다양한 에러 상황을 열거형으로 정의하기.
    - **Implementing `Error` Trait:** 표준 `std::error::Error` 트레이트 구현하기.
    - **`Display` and `Debug`:** 에러 메시지를 사용자에게 친숙하게 보여주기.
- **실습 지시:**
    - `DatabaseError`와 `NetworkError`를 포함하는 `AppError` 열거형 만들기.

### 14:45~16:00: Error Handling Libraries (anyhow, thiserror)
- **목표:** 실무에서 사용되는 에러 처리 라이브러리의 차이점과 사용법을 익힌다.
- **주요 내용:**
    - **`thiserror`:** 라이브러리 개발자를 위한 에러 정의 도구.
    - **`anyhow`:** 애플리케이션 개발자를 위한 간편한 에러 처리 도구.
- **실습 지시:**
    - `thiserror`를 사용하여 커스텀 에러를 정의하고, `anyhow`를 사용하여 애플리케이션 상위 레벨에서 에러를 처리하기.

### 16:10~18:00: 랩 (Lab) 및 과제
- **목표:** 실제적인 에러 처리 로직을 구축한다.
- **활동:**
    - **파일 처리 시스템 구현:**
        - 파일 읽기, 파싱, 데이터 유효성 검사 과정에서 발생하는 다양한 에러를 `Result`로 관리하고, `?` 연산자를 통해 깔끔하게 처리하는 프로그램 작성.
    - **과제: 견고한 CLI 도구 만들기**
        - 사용자 입력값 검증부터 파일 입출력까지 모든 단계에서 발생할 수 있는 에러를 커스텀 에러 타입으로 정의하고, 사용자에게 친절한 에러 메시지를 출력하는 CLI 도구 완성하기.
