# Day 6. 소유권(Ownership)과 빌림(Borrowing) 기초

## 1. 강의 개요
- **일자:** Day 6
- **목표:** Rust의 가장 핵심적인 개념인 소유권(Ownership)과 빌림(Borrowing)의 원리를 이해하고, 이를 활용한 안전한 메모리 관리 방법을 익힌다.

---

## 2. 시간대별 세션

### 09:00~10:30: 소유권(Ownership)의 세 가지 규칙
- **목표:** Rust가 가비지 컬렉터 없이 어떻게 메모리 안전성을 보장하는지 이해한다.
- **주요 내용:**
    - **Ownership의 3대 규칙:**
        1. Rust의 각 값은 '소유자(owner)'라고 불리는 변수를 갖는다.
        2. 한 번에 하나의 소유자만 존재할 수 있다.
        3. 소유자가 스코프(scope)를 벗어나면, 값은 드롭(drop)된다.
    - **Move semantics:** 힙(Heap)에 할당된 데이터를 가진 변수를 다른 변수에 할당할 때 발생하는 '이동' 현상 이해하기.
    - **Copy trait:** 스택(Stack)에 저장된 단순 타입(정수, 불리언 등)의 복사 동작 이해하기.
- **실습 지시:**
    - `String` 변수를 다른 변수에 할당한 후, 원래 변수를 사용했을 때 발생하는 컴파일 에러 확인하기.
    - 스칼라 타입(i32)을 다른 변수에 할당하고 두 변수 모두 사용해보기.

### 10:45~12:00: 참조와 빌림(References and Borrowing)
- **목표:** 소유권을 넘기지 않고도 데이터를 사용할 수 있는 방법을 배운다.
- **주요 내용:**
    - **References (&):** 값을 소유하지 않고 참조하는 법.
    - **Immutable Borrowing:** 여러 개의 불변 참조를 동시에 가질 수 있는 규칙.
    - **Mutable Borrowing (&mut):** 단 하나의 가변 참조만 가질 수 있는 규칙 (데이터 경합 방지).
    - **The Rules of Borrowing:** "동시에 여러 개의 불변 참조 OR 단 하나의 가변 참조" 규칙의 중요성.
- **실습 지시:**
    - 함수에 `&String`을 전달하여 소유권 이동 없이 값 읽기.
    - 함수에 `&mut String`을 전달하여 값을 수정하기.
    - 불변 참조와 가변 참조를 동시에 사용하려고 할 때 발생하는 컴파일 에러 관찰하기.

### 13:00~14:30: 슬라이스(Slices)와 참조의 결합
- **목표:** 데이터의 일부를 안전하게 참조하는 법을 익힌다.
- **주요 내용:**
    - **String Slices (&str):** 문자열의 특정 범위를 가리키는 참조.
    - **Slice Type:** `&[T]` 형태의 슬라이스 개념.
    - 슬라이스가 소유권 규칙(Borrowing)에 어떻게 적용되는지 확인.
- **실습 지시:**
    - 문자열의 일부를 슬라이스로 추출하여 함수 인자로 전달하기.
    - 배열의 일부를 슬라이스로 만들어 함수에서 처리하기.

### 14:45~16:00: 데이터 경합(Data Race) 방지와 컴파일러의 역할
- **목표:** Rust의 빌림 검사기(Borrow Checker)가 어떻게 런타임 에러를 컴파일 타임으로 끌어올리는지 이해한다.
- **주요 내용:**
    - **Data Race란 무엇인가?** (동시 접근 + 쓰기 작업의 위험성).
    - **Borrow Checker의 작동 원리:** 컴파일 타임에 참조의 유효성을 검사하는 과정.
    - 에러 메시지를 통한 해결 방법: "cannot borrow as mutable because it is also borrowed as immutable".
- **실습 지시:**
    - 의도적으로 데이터 경합이 발생할 수 있는 코드(예: 루프 내 참조와 수정)를 작성하고 컴파일러의 가이드 읽기.

### 16:10~18:00: 랩 (Lab) 및 과제
- **목표:** 소유권과 빌림을 활용하여 안전한 데이터 처리 로직을 구현한다.
- **활동:**
    - **데이터 처리 파이프라인 구현:**
        - 문자열 리스트(`Vec<String>`)를 입력받아, 특정 패턴을 가진 단어만 추출하여 새로운 리스트를 만드는 함수 작성.
        - 이때 소유권을 이동시키지 않고 참조(`&str`)만을 사용하여 효율적으로 처리하기.
    - **과제: 안전한 공유 저장소(Simple Shared Store) 설계**
        - 특정 데이터를 저장하고, 읽기(불변 참조)와 수정(가변 참조)이 가능한 구조체 작성.
        - 빌림 규칙을 위반하는 상황(읽는 중에 수정 시도 등)을 코드로 재현해보고 컴파일러 메시지 기록하기.
