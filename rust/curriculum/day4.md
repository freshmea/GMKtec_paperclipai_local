# Day 4. 컬렉션과 모듈 기초

## 1. 강의 개요
- **일자:** Day 4
- **목표:** 동적 배열(`Vec<T>`)을 배우고, 프로젝트를 여러 파일로 나누어 관리하는 모듈 시스템을 익힌다.

---

## 2. 시간대별 세션

### 09:00~10:30: `Vec<T>` 기초와 반복 처리
- **목표:** 런타임에 크기가 변하는 동적 배열인 `Vec`을 이해하고 사용한다.
- **주요 내용:**
    - `Vec<T>` 정의 및 생성 (`vec![]` 매크로)
    - 주요 메서드: `push`, `pop`, `len`, `is_empty`, `get` (Option 반환)
    - 벡터의 소유권과 이동(Move)
    - 벡터 순회: `for item in &v` (참조) vs `for item in v` (소유권 이동)
- **실습 지시:**
    - `Vec`에 요소를 추가하고 삭제해보기
    - `get` 메서드를 사용하여 안전하게 인덱스 접근하기 (Panic 방지)

### 10:45~12:00: 모듈 시스템 (Module System) 기초
- **목표:** 코드를 논리적으로 분리하고 관리하는 Rust의 모듈 구조를 이해한다.
- **주요 내용:**
    - `mod` 키워드: 모듈 선언
    - `use` 키워드: 경로 단축
    - `pub` 키워드: 가시성(Visibility) 제어 (Private vs Public)
    - 파일 기반 모듈: `mod.rs` (구식) vs `filename.rs` (현대적) 구조
- **실습 지시:**
    - 하나의 파일 내에서 `mod`를 사용하여 기능 분리해보기
    - 다른 파일에 정의된 함수를 `use`로 가져와 사용하기

### 13:00~14:30: 다중 파일 구조로 리팩터링
- **목표:** 작은 프로젝트를 실제 규모 있는 폴더/파일 구조로 확장한다.
- **주요 내용:**
    - `src/main.rs`와 `src/lib.rs`의 역할
    - 디렉토리 구조를 이용한 계층적 모듈 설계
    - `crate::` 경로를 이용한 절대 경로 접근
- **실습 지시:**
    - Day 1~3에서 만든 간단한 도구(예: 계산기)를 `math` 모듈과 `main` 모듈로 분리하기

### 14:45~16:00: Rust 개발 루틴 (Lint & Format)
- **목표:** 표준 도구를 사용하여 코드 품질을 유지하는 습관을 들인다.
- **주요 내용:**
    - `cargo fmt`: 코드 스타일 자동 교정
    - `cargo clippy`: 정적 분석을 통한 잠재적 버그 및 개선안 제안
    - `cargo check`: 컴파일 가능 여부 빠른 확인
- **실습 지시:**
    - 일부러 스타일을 엉망으로 작성한 후 `cargo fmt` 실행하기
    - 비효율적인 코드를 작성한 후 `cargo clippy`가 주는 조언 확인하기

### 16:10~18:00: 랩 (Lab) 및 과제
- **목표:** 모듈화된 구조를 갖춘 CLI 프로그램을 작성한다.
- **활동:**
    - **전날 과제 재정리:** Day 3의 통계 프로그램을 모듈 구조로 리팩터링
    - **과제: 간단한 메모리 기반 TODO CLI 초안**
        - `Todo` 구조체와 `TodoList` 구조체를 별도 모듈로 분리
        - `Vec<Todo>`를 사용하여 항목 추가, 목록 출력 기능 구현
        - `pub` 키워드를 적절히 사용하여 기능 노출 범위 결정
