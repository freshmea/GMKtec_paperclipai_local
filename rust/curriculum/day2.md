## 2. 시간대별 세션

### 09:00~10:30: 함수와 표현식 vs 문장
- **목표:** 함수의 기본 구조와 Rust의 핵심 개념인 '표현식(Expression)'을 이해한다.
- **주요 내용:**
    - 함수의 선언: `fn` 키워드, 매개변수(Parameters), 반환 타입(Return Type)
    - **Expression vs Statement:**
        - Statement: 값을 생성하지 않고 동작만 수행 (예: `let x = 5;`)
        - Expression: 값을 계산하여 반환 (예: `5 + 5`, `if { ... }`)
    - 함수의 암시적 반환 (Implicit Return): 세미콜론(`;`) 유무의 차이
- **실습 지시:**
    - 간단한 더하기 함수 작성하기
    - 마지막 라인에 세미콜론을 뺐을 때와 넣었을 때의 차이 확인하기

### 10:45~12:00: 제어 흐름 (Control Flow)
- **목표:** 프로그래밍의 기본 제어 구조를 Rust 문법으로 익힌다.
- **주요 내용:**
    - `if` 표현식: 조건문을 값으로 사용하는 법
    - 루프(Loops):
        - `loop`: 무한 루프 및 `break` 사용
        - `while`: 조건부 루프
        - `for`: 이터레이터 기반 루프 (범위 `1..5`, `1..=5` 포함)
- **실습 지시:**
    - `loop`를 사용하여 1부터 10까지 출력하기
    - `for` 루프를 사용하여 배열의 요소 순회하기

### 13:00~14:30: FizzBuzz 및 숫자 맞히기 실습
- **목표:** 제어 흐름을 조합하여 실제 논리 문제를 해결한다.
- **주요 내용:**
    - FizzBuzz 알고리즘 구현 (3의 배수, 5의 배수, 공배수 처리)
    - `match` 문을 활용한 깔끔한 제어 흐름 설계 (맛보기)
- **실습 지시:**
    - `FizzBuzz` 프로그램을 작성하여 콘솔에 출력하기
    - `if/else if/else` 구조를 사용하여 조건 분기 연습하기

### 14:45~16:00: 코드 읽기 훈련과 컴파일 에러 해석
- **목표:** 복잡한 코드의 흐름을 파악하고, 컴파일러의 도움을 받는 법을 배운다.
- **주요 내용:**
    - 다양한 제어 흐름이 섞인 코드 분석하기
    - **컴파일 에러 읽기:** "mismatched types", "expected one expression but found..." 등의 메시지 의미 파악
    - `cargo check`를 통한 빠른 피드백 루프
- **실습 지시:**
    - 의도적으로 잘못된 타입의 값을 반환하는 함수를 만들고 에러 메시지 읽어보기

### 16:10~18:00: 랩 (Lab) 및 과제
- **목표:** 실습 내용을 종합하여 작은 도구를 완성한다.
- **활동:**
    - **숫자 맞히기 게임 개선:** 
        - 사용자 입력을 받아(간단하게 하드코딩 또는 `std::io` 사용) 숫자를 맞히는 로직 구현
        - `loop`와 `break`를 활용하여 정답을 맞힐 때까지 반복
    - **과제: 단위 변환기 CLI 작성**
        - Celsius $\leftrightarrow$ Fahrenheit, Meter $\leftrightarrow$ Foot 등 간단한 변환 기능
        - 함수를 사용하여 각 변환 로직 분리
        - `match` 또는 `if`를 사용하여 변환 모드 선택
