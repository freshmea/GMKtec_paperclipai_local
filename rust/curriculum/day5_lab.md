# Day 5. 1주차 정리와 미니 과제

## 1. 강의 개요
- **일자:** Day 5
- **목표:** 1주차에 배운 모든 기초 개념을 통합하고, 실제 동작하는 완성도 있는 프로그램을 구현한다.

---

## 2. 시간대별 세션

### 09:00~10:30: 1주차 개념 총정리 및 오류 리뷰
- **목표:** 1주차 동안 학습한 핵심 개념을 복습하고, 공통적으로 발생했던 오류 패턴을 분석한다.
- **주요 내용:**
    - **Review:** 변수(Immutability), 타입(Scalar/Compound), 함수, 제어 흐름, 문자열(`String`/`&str`), 컬렉션(`Vec`), 모듈 시스템
    - **Error Pattern Analysis:** 
        - Ownership/Borrowing의 기초적인 위반 사례 (다음 주차 예고)
        - 타입 불일치(Mismatched types) 및 Casting 오류
        - 문자열 슬라이싱 시 유니코드 패닉
- **활동:**
    - 1주차 핵심 키워드 퀴즈

### 10:45~12:00: 디버깅 전략 및 문서 읽기
- **목표:** 문제가 발생했을 때 스스로 해결할 수 있는 능력을 기른다.
- **주요 내용:**
    - **Debugging Tools:**
        - `println!` 매크로를 활용한 값 출력
        - `dbg!` 매크로: 파일명, 라인 번호와 함께 값 출력 (강력 추천!)
    - **Reading Documentation:**
        - `rustdoc` 활용법
        - `docs.rs`에서 라이브러리 문서 찾는 법
        - 공식 Rust Book 읽는 법
- **실습 지시:**
    - 복잡한 변수 값을 `dbg!`로 출력하여 흐름 추적하기

### 13:00~14:30: 미니 과제 구현 (Final Project of Week 1)
- **목표:** 지금까지 배운 내용을 총동원하여 하나의 완성된 프로그램을 만든다.
- **주요 내용:**
    - **선택 과제 (둘 중 하나 수행):**
        1. **강화된 계산기:** 사칙연산 외에 제곱, 루트 등을 포함하고, 모듈화된 구조를 가짐
        2. **TODO CLI (v1):** 메모리 기반으로 항목 추가, 삭제, 목록 보기가 가능한 도구
    - **요구 사항:**
        - 최소 2개 이상의 모듈로 분리할 것
        - `match` 또는 `if let`을 사용하여 제어 흐름을 구현할 것
        - `cargo check`와 `cargo fmt`를 통과할 것

### 14:45~16:00: 페어 리뷰 및 리팩터링
- **목표:** 타인의 코드를 보고 내 코드를 개선하는 경험을 한다.
- **활동:**
    - **Peer Review:** 2인 1조로 서로의 코드를 읽고 개선점 제안 (코드 스타일, 논리적 명확성)
    - **Refactoring:** 리뷰 내용을 바탕도 코드 구조 개선 (함수 추출, 변수명 변경 등)

### 16:10~18:00: 주간 점검 및 마무리
- **목표:** 학습 성과를 확인하고 다음 주차를 준비한다.
- **활동:**
    - **주간 퀴즈:** 1주차 핵심 개념 테스트
    - **과제 제출:** 완성된 코드를 `code/` 디렉토리에 정리
    - **Next Step Preview:** 2주차의 핵심 주제인 **Ownership, Borrowing, Lifetime** 맛보기 설명
