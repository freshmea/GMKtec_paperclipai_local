# ROS vs ROS 2: 비교 개요

Robot Operating System (ROS)은 다양한 로봇 플랫폼에서 복잡하고 견고한 로봇 동작을 생성하는 작업을 단순화하는 도구, 라이브러리 및 관례의 모음입니다.

## ROS (Robot Operating System 1)
본래 연구용으로 설계된 ROS 1은 중앙 집중식 아키텍처를 기반으로 구축되었습니다.

- **아키텍처:** 노드 간 통신을 조정하기 위해 ROS Master를 사용합니다.
- **통신:** 주로 발견(discovery)을 위해 중앙 집중식 마스터에 의존합니다.
- **강점:** 기존 패키지의 방대한 생태계, 연구 분야에 잘 정립됨, 광범위한 커뮤니티 지원.
- **약점:** 단일 장애점(마스터) 존재, 실시간 애플리케이션 지원 제한적, 다중 로봇 시스템에 덜 견고함, 자원이 제한된 환경에 배포하기 어려움.

## ROS 2 (Robot Operating System 2)
ROS 1의 한계를 해결하기 위해 설계된 ROS 2는 생산 단계의 로보틱스를 위해 구축되었습니다.

- **아키텍처:** 발견 및 통신을 위해 Data Distribution Service (DDS)를 사용하는 분산형 아키텍처입니다.
- **통신:** 중앙 마스터 없이 피어 투 피어(P2P) 통신을 수행하여 더 견고하고 확장 가능합니다.
- **강점:**
    - **실시간 지원:** 실시간 성능과 결정론적 실행을 위해 설계되었습니다.
    - **다중 로봇 시스템:** 동일한 네트워크에서 작동하는 여러 로봇에 대한 지원이 향상되었습니다.
    - **신뢰성:** 분산형 DDS 아키텍처 덕분에 단일 장애점이 없습니다.
    - **보안:** 안전한 통신을 위한 내장 보안 기능(SROS2)을 제공합니다.
    - **엣지/임베디드 배포:** 자원이 제한된 하드웨어 및 산업용 애플리케이션에 더 적합합니다.
- **약점:** 학습 곡선이 더 가파르며, 일부 레거시 ROS 1 패키지에 직접적인 ROS 2 대응물이 없을 수 있습니다 (호환성 레이어는 존재함).

## 비교 표

| 기능 | ROS 1 | ROS 2 |
| :--- | :--- | :--- |
| **발견 (Discovery)** | 중앙 집중식 (ROS Master) | 분산형 (DDS) |
| **실시간성** | 제한적/네이티브 아님 | 네이티브 지원 |
| **다중 로봇** | 어려움/우회 방법 필요 | 내장 지원 |
| **보안** | 최소 수준/추가 기능 필요 | 통합됨 (SROS2) |
| **상용화 준비** | 주로 연구용 | 생산용으로 설계됨 |

## 사용 사례 요약
- **ROS 1을 사용하는 경우:** 기존 ROS 1 패키지가 있는 레거시 연구 프로젝트를 진행 중이며, 실시간 제어나 고수준의 상용 배포가 필요하지 않은 경우.
- **ROS 2를 사용하는 경우:** 새로운 프로젝트를 시작하거나, 실시간 기능이 필요하거나, 여러 대의 로봇을 관리해야 하거나, 생산 단계의 산업용 또는 상업용 애플리케이션을 목표로 하는 경우.
