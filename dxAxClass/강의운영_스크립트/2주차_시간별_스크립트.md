# 2주차 시간별 강의운영 스크립트

## 6일차

- 일일 목표: 1주차 요구사항을 프롬프트 구조로 전환한다.
- 종료 산출물: 기본 프롬프트 템플릿, 업무별 프롬프트 초안

### 1교시 2주차 오리엔테이션
- 스크립트: "오늘부터는 AI를 실제로 움직입니다. 지난주 문서를 입력값으로 삼아 재현 가능한 프롬프트를 만들겠습니다."
- 참조할 내용: 1주차 산출물 묶음, 2주차 목표, 결과물 생산 흐름
- 수업 팁: 요구사항 정의서를 다시 펼치게 하고 빈손으로 프롬프트를 쓰지 않게 한다.

### 2교시 생성형 AI 동작 원리 이해
- 스크립트: "프롬프트를 잘 쓰려면 AI가 어떻게 답을 만드는지 알아야 합니다. 토큰과 컨텍스트 관점에서 왜 결과가 흔들리는지 보겠습니다."
- 참조할 내용: 토큰, 컨텍스트, 예측 구조, 환각 가능성
- 수업 팁: 기술 설명은 짧게 하고 결과 품질과 직접 연결되는 부분만 남긴다.

### 3교시 프롬프트 설계의 기본 원칙
- 스크립트: "좋은 프롬프트는 질문이 아니라 업무 지시서입니다. 명확성, 맥락, 제약조건이 기본 골격입니다."
- 참조할 내용: 역할, 목적, 입력, 절차, 출력 형식
- 수업 팁: 단문 예시보다 잘못된 프롬프트를 고치는 방식이 이해가 빠르다.

### 4교시 프롬프트 구조화 기법
- 스크립트: "같은 일을 맡겨도 구조가 있으면 품질이 안정됩니다. 역할과 절차, 출력 형식을 고정해 보겠습니다."
- 참조할 내용: 구조화 프롬프트 템플릿, 단계형 지시문
- 수업 팁: 템플릿을 그대로 외우게 하지 말고 자기 업무에 맞게 문구를 바꾸게 한다.

### 5교시 기획 업무 적용 실습
- 스크립트: "이제 기획 업무를 예시로 프롬프트를 적용해 봅니다. 아이디어 발산과 초안 구조를 분리해서 실습하겠습니다."
- 참조할 내용: 기획 초안 프롬프트, 입력 자료, 출력 예시
- 수업 팁: 결과가 길어지면 `결론 3개만` 같은 출력 제약을 추가하게 한다.

### 6교시 보고 업무 적용 실습
- 스크립트: "보고 업무는 누가 읽는지에 따라 문장이 달라집니다. 목적과 독자를 분명히 적었는지 다시 확인합시다."
- 참조할 내용: 요약 보고, 현황 보고, 제안 보고 예시
- 수업 팁: 같은 자료로 임원용과 실무자용 두 버전을 비교하게 하면 차이가 선명하다.

### 7교시 회의 업무 적용 실습
- 스크립트: "회의 업무는 전후 맥락이 중요합니다. 아젠다, 회의록, 후속 조치를 하나의 흐름으로 묶어 보겠습니다."
- 참조할 내용: 회의 아젠다, 회의록, 액션 아이템 템플릿
- 수업 팁: `회의 전-중-후`로 구분해서 프롬프트를 만들게 하면 재사용성이 높다.

### 8교시 실무 적용 Q&A 및 정리
- 스크립트: "오늘 만든 프롬프트는 내일 문서 초안 생산의 엔진입니다. 템플릿과 실제 적용본을 둘 다 저장해 주세요."
- 참조할 내용: 프롬프트 템플릿 저장 규칙, 내일 입력 자료
- 수업 팁: 잘 만든 프롬프트보다 수정 이유가 명확한 프롬프트를 공유하면 학습 효과가 좋다.

## 7일차

- 일일 목표: 문서 유형별 프롬프트와 검수 기준을 만들고 결과물을 개선한다.
- 종료 산출물: 문서 프롬프트 명세, 유형별 문서 초안, 개선본

### 1교시 비즈니스 문서의 핵심 구조
- 스크립트: "문서는 다 비슷해 보여도 설득 구조가 다릅니다. 오늘은 문서 유형별 논리 흐름을 먼저 구분하겠습니다."
- 참조할 내용: 이메일, 보고서, 제안서 구조 비교
- 수업 팁: 문서 목적을 한 문장으로 먼저 쓰게 하면 구조 선택이 쉬워진다.

### 2교시 AI를 위한 문서 명세 설계
- 스크립트: "좋은 초안은 좋은 입력 조건에서 나옵니다. 독자, 목적, 톤, 분량을 명세로 고정하겠습니다."
- 참조할 내용: 문서 명세 항목, 입력 조건 설계법
- 수업 팁: `누구에게 무엇을 하게 할 문서인지`를 빠뜨리지 않게 한다.

### 3교시 유형별 문서 초안 생성 기법
- 스크립트: "이메일, 보고서, 제안서는 프롬프트도 달라야 합니다. 문서별로 무엇을 강조할지 나눠서 설계하겠습니다."
- 참조할 내용: 유형별 프롬프트 예시, 출력 형식 차이
- 수업 팁: 한 프롬프트로 모든 문서를 만들려는 시도를 막는다.

### 4교시 [실습] AI 활용 문서 초안 생성
- 스크립트: "업무 시나리오를 선택하고 초안을 생성해 보겠습니다. 지금 단계에서는 완성보다 뼈대를 만드는 것이 목표입니다."
- 참조할 내용: 업무 시나리오 카드, 초안 생성 프롬프트
- 수업 팁: 초안과 최종본을 분리 저장하게 해야 개선 학습이 남는다.

### 5교시 결과 보완 및 개선 방법론
- 스크립트: "생성은 시작일 뿐이고 완성도는 검수에서 나옵니다. 논리, 사실, 완성도 기준으로 다시 보겠습니다."
- 참조할 내용: 검수 체크리스트, 사실성 점검, 논리 흐름
- 수업 팁: 개인 취향 교정보다 목적 적합성 교정에 집중한다.

### 6교시 문장 교정 및 톤앤매너 조정
- 스크립트: "비즈니스 문서는 맞는 내용이어도 어조가 맞지 않으면 쓰기 어렵습니다. 같은 내용의 표현 강도를 조정해 보겠습니다."
- 참조할 내용: 톤앤매너 예시, 정중도와 명확성 조정법
- 수업 팁: 전부 공손하게 만드는 것보다 상황별 어조 선택을 보여 준다.

### 7교시 [실습] 문서 완성도 제고
- 스크립트: "이제 초안을 완성형 문서로 고도화합니다. 수정 이유를 한 줄씩 남기면서 바꿔 주세요."
- 참조할 내용: 수정 전후 비교 기준, 개선 로그 양식
- 수업 팁: 문서 수준 차이가 크면 이메일 트랙과 보고서 트랙으로 분리 운영한다.

### 8교시 7일차 종합 리뷰 및 피드백
- 스크립트: "내일은 긴 자료를 요약하고 재구성합니다. 오늘 만든 문서 중 하나를 요약 대상 자료로 저장해 두세요."
- 참조할 내용: 문서 초안과 완성본, 내일 요약용 자료
- 수업 팁: 8일차 입력 자료를 오늘 바로 지정해 두면 다음 날 이탈이 줄어든다.

## 8일차

- 일일 목표: 긴 자료를 목적 중심 요약과 인사이트 메모로 전환한다.
- 종료 산출물: 요약 프롬프트, 실무 문서 요약본, 인사이트 메모

### 1교시 정보 수집·요약의 패러다임 변화
- 스크립트: "정보가 많을수록 중요한 것은 다 읽는 능력이 아니라 필요한 기준으로 다시 정리하는 능력입니다."
- 참조할 내용: 정보 과부하, 요약 역량의 의미, 요약과 판단의 차이
- 수업 팁: 요약을 단순 축약으로 이해하지 않게 목적을 먼저 제시한다.

### 2교시 생성형 AI 정보 수집 방식 이해
- 스크립트: "정보 수집 도구는 많지만, 중요한 것은 소스의 성격을 알고 쓰는 것입니다. 오늘은 수집과 검증을 함께 보겠습니다."
- 참조할 내용: 문서, 웹, 회의 기록, 영상·음성 소스 구분
- 수업 팁: 출처 확인이 필요한 정보와 아닌 정보를 구분해 준다.

### 3교시 효과적인 요약 프롬프트 구조
- 스크립트: "좋은 요약은 대상, 목적, 분량 기준이 명확합니다. 누구를 위한 요약인지 먼저 정하겠습니다."
- 참조할 내용: 대상 독자, 목적, 분량, 출력 형식
- 수업 팁: 임원용과 실무자용 요약의 차이를 예시로 먼저 보여 준다.

### 4교시 [실습] AI 활용 텍스트 요약
- 스크립트: "같은 자료를 서로 다른 목적에 맞게 요약해 보겠습니다. 요약문과 핵심 bullet을 분리해서 만들어 주세요."
- 참조할 내용: 긴 문서 샘플, 요약 프롬프트, 길이 제약
- 수업 팁: 과도하게 짧아져서 맥락이 날아가지 않도록 최소 포함 항목을 지정한다.

### 5교시 멀티모달 요약 및 정리
- 스크립트: "요약 대상은 문서만이 아닙니다. 영상과 음성도 업무 입력값이 되므로 정리 기준을 확장해 보겠습니다."
- 참조할 내용: 영상·음성 요약 사례, 멀티모달 입력 구조
- 수업 팁: 기능 소개보다 텍스트로 환원되는 결과 구조를 강조한다.

### 6교시 핵심 내용 정리 및 지식화 훈련
- 스크립트: "요약 결과가 가치 있으려면 다시 의사결정용 메모로 바뀌어야 합니다. 핵심 내용과 시사점을 분리해 보겠습니다."
- 참조할 내용: 인사이트 메모 양식, 핵심 내용과 시사점 구분
- 수업 팁: 메모에는 반드시 `그래서 무엇을 할 것인가` 한 줄을 넣게 한다.

### 7교시 [실습] 실무 문서 핵심 정리
- 스크립트: "각자 실제 문서를 골라 요약본과 인사이트 메모를 완성하겠습니다. 출력 대상 독자를 끝까지 유지해 주세요."
- 참조할 내용: 실무 문서, 요약본, 인사이트 메모 템플릿
- 수업 팁: 자료 난도가 높으면 공통 예제로 한 번 시연한 뒤 개인 자료로 넘어간다.

### 8교시 8일차 종합 정리 및 리뷰
- 스크립트: "내일은 숫자와 표를 해석하는 관점으로 넘어갑니다. 오늘 만든 인사이트 메모는 데이터 해석 보고의 말하기 기준이 됩니다."
- 참조할 내용: 요약 산출물, 인사이트 메모, 내일 연결 포인트
- 수업 팁: 보고용 언어와 요약용 언어의 차이를 마지막에 짧게 정리해 준다.

## 9일차

- 일일 목표: 데이터 리터러시를 익히고 AI 분석 결과를 해석하는 언어를 만든다.
- 종료 산출물: 기초 데이터 분석 결과, 인사이트 정리 메모

### 1교시 데이터 분석의 필요성
- 스크립트: "데이터 분석의 핵심은 계산보다 질문입니다. 오늘은 숫자 뒤의 맥락을 읽는 관점을 만들겠습니다."
- 참조할 내용: DX·AX에서 데이터 해석의 의미, 의사결정 연결
- 수업 팁: 데이터에 약한 학습자도 들어올 수 있게 숫자보다 질문부터 시작한다.

### 2교시 데이터 분석 기본 개념
- 스크립트: "데이터 유형과 분석 목적을 구분하면 어떤 질문을 던져야 할지 보입니다."
- 참조할 내용: 정형·비정형 데이터, 기술통계, 비교, 추세
- 수업 팁: 용어 암기보다 실제 사례에 바로 붙여 설명한다.

### 3교시 분석 흐름 및 해석 방법
- 스크립트: "수집, 정제, 분석, 시각화는 따로 노는 단계가 아닙니다. 해석을 위해 어떤 정리가 필요한지 보겠습니다."
- 참조할 내용: 분석 흐름, 데이터 정제, 해석 단계
- 수업 팁: 결과 수치만 보지 말고 데이터가 어떻게 만들어졌는지도 묻게 한다.

### 4교시 데이터 리터러시 역량
- 스크립트: "좋은 해석은 숫자를 단정하지 않습니다. 가능한 설명을 열어 두고 근거를 찾는 태도가 중요합니다."
- 참조할 내용: 가설 언어, 맥락 읽기, 성급한 일반화 방지
- 수업 팁: `왜 그렇다`보다 `그럴 수 있다` 표현을 쓰게 지도한다.

### 5교시 AI 기반 데이터 분석 도구
- 스크립트: "AI 분석 도구는 패턴을 제시하지만, 최종 해석 책임은 사람에게 있습니다. 도구의 역할을 과장하지 맙시다."
- 참조할 내용: 주요 AI 분석 도구, 장단점, 검수 필요성
- 수업 팁: 도구 소개는 짧게, 결과 검수 기준은 길게 다룬다.

### 6교시 [실습] 간단한 데이터 분석
- 스크립트: "샘플 데이터를 기반으로 기초 분석을 해 보겠습니다. 오늘은 복잡한 모델보다 읽을 수 있는 결과를 만드는 데 집중합니다."
- 참조할 내용: 샘플 데이터, 분석 질문 리스트, 기본 지표
