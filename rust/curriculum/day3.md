# Day 3. 소유 타입과 문자열 기초

## 1. 강의 개요
- **일자:** Day 3
- **목표:** Rust의 핵심인 데이터 타입(정수/실수/불리언/문자)과 문자열(`String`, `&str`)의 차이를 명확히 이해한다.

---

## 2. 시간대별 세션

### 09:00~10:30: 기본 타입 심화 (Casting 및 주의점)
- **목표:** 스칼라 타입의 세부 사항과 타입 변환의 엄격함을 이해한다.
- **주요 내용:**
    - 정수형(`i8`~`i128`, `u8`~`u128`)과 부동 소수점형(`f32`, `f64`)의 범위와 용도
    - **Type Casting:** `as` 키워드를 사용한 명시적 변환
    - Casting 시 발생할 수 있는 데이터 손실(Overflow, Truncation) 사례
    - `bool` 타입의 특성
- **실습 지시:**
    - 큰 타입에서 작은 타입으로 `as`를 사용하여 변환해보고 값의 변화 관찰하기
    - 부동 소수점과 정수 간의 변환 테스트

### 10:45~12:00: 문자열의 두 얼굴: `String` vs `&str`
- **목표:** Rust에서 가장 혼란스러운 문자열 처리의 기초를 잡는다.
- **주요 내용:**
    - **`String`:** Heap에 할당된, 크기 조절이 가능한 소유권이 있는 문자열 (Owned string)
    - **`&str`:** 문자열 슬라이스, 특정 메모리 영역을 가리키는 참조 (String slice)
    - 문자열 생성 방법: `String::from()`, `.to_string()`, `"literal"`
- **실습 지시:**
    - `String`을 생성하고 `&str`로 변환하기
    - `&str` 변수를 `String`으로 변환하기

### 13:00~14:30: 문자열 조작 및 UTF-8 주의점
- **목표:** 문자열을 자르고(slicing) 다루는 법과 유니코드의 특성을 이해한다.
- **주요 내용:**
    - **String Slicing:** 인덱스를 이용한 부분 문자열 추출
    - **UTF-8 Encoding:** Rust 문자열은 항상 유효한 UTF-8임을 명시
    - **Indexing의 위험성:** 한글(멀티바이트) 슬라이싱 시 발생하는 패닉(Panic) 사례
    - `.chars()`, `.bytes()` 이터레이터 활용
- **실습 지시:**
    - 영어 문자열 슬라이싱과 한글 문자열 슬라이싱 비교하기 (에러 유도)
    - `.chars().collect()` 등을 사용하여 문자열 조작해보기

### 14:45~16:00: 배열과 슬라이스 기반 데이터 처리
- **목표:** 고정 크기 데이터 집합을 효율적으로 다루는 법을 익힌다.
- **주요 내용:**
    - 배열(Array)의 특징: 고정 크기, 동일 타입
    - 슬라이스(Slice)의 개념: 배열의 일부를 참조하는 방법 (`&arr[1..3]`)
    - 함수 인자로 슬라이스 전달하기 (유연한 인터페이스 설계)
- **실습 지시:**
    - 배열을 생성하고 특정 범위를 슬라이스로 만들어 함수에 전달하기
    - 슬라이스를 이용해 배열의 합계나 평균 구하기

### 16:10~18:00: 랩 (Lab) 및 과제
- **목표:** 문자열과 기본 타입을 조합하여 실습한다.
- **활동:**
    - **문자열 정리 도구 작성:**
        - 입력된 문자열의 공백 제거, 대소문자 변환, 특정 문자 교체 기능
    - **과제: 입력 텍스트 통계 프로그램**
        - 사용자로부터 문장(또는 파일 내용)을 입력받음
        - 단어 수, 문자 수, 특정 단어의 빈도수를 계산하여 출력
        - `String`과 `&str`을 적절히 혼용하여 구현
