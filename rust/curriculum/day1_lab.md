# Day 1. Rust 소개와 개발 환경

## 1. 강의 개요
- **일자:** Day 1
- **목표:** Rust의 철학 이해, 개발 환경 구축, 기본 문법(변수, 타입) 익히기

---

## 2. 시간대별 세션

### 09:00~10:30: 과정 오리엔테이션 & Rust의 철학
- **목표:** Rust가 왜 만들어졌는지, 어떤 문제를 해결하려 하는지 이해한다.
- **주요 내용:**
    - Rust의 핵심 가치: **Memory Safety without Garbage Collector**
    - **Zero-cost Abstractions:** 추상화가 성능에 영향을 주지 않음
    - **Fearless Concurrency:** 컴파일 타임에 데이터 경합(Data Race) 방지
    - 메모리 관리 모델 맛보기: Stack vs Heap 개념 설명
- **데모 포인트:**
    - C/C++에서의 메모리 버그(Use-after-free, Double free) 사례 제시 (설명 위주)
    - Rust가 이를 어떻게 컴파일 타임에 잡아내는지 개념적 예시

### 10:45~12:00: 개발 환경 구축 및 Cargo 기초
- **목표:** Rust 개발 도구를 설치하고 첫 프로젝트를 생성/실행한다.
- **주요 내용:**
    - `rustup`: Rust 툴체인 관리자
    - `rustc`: Rust 컴파일러
    - `cargo`: Rust의 빌드 시스템 및 패키지 매니저 (가장 중요!)
    - `cargo new <name>`: 새 프로젝트 생성
    - `cargo run`: 빌드 및 실행
    - `cargo check`: 빠른 컴파일 확인 (빌드 없이 체크만)
- **실습 지시:**
    1. `rustup` 설치 확인 (`rustup --version`)
    2. `cargo new hello_rust` 명령으로 프로젝트 생성
    3. `src/main.rs` 파일 확인 및 `cargo run` 실행

### 13:00~14:30: 변수, 상수, Shadowing, 스칼라 타입
- **목표:** Rust의 변수 선언 방식과 기본 데이터 타입을 익힌다.
- **주요 내용:**
    - **Immutability (불변성):** `let` vs `let mut`
    - **Constants (상수):** `const`와 `static`의 차이
    - **Shadowing (섀도잉):** 동일한 이름의 변수를 재선언하여 타입/값 변경하기
    - **Scalar Types:**
        - 정수형 (`i32`, `u32`, `i64` 등)
        - 부동 소수점형 (`f32`, `f64`)
        - 불리언 (`bool`)
        - 문자 (`char`)
- **실습 지시:**
    - 불변 변수에 값을 재할당 시도해보고 에러 확인하기
    - `let mut`를 사용하여 가변 변수 만들기
    - 섀도잉을 사용하여 `String` 타입을 `usize`로 변환해보기

### 14:45~16:00: 튜플, 배열, 기본 출력 및 디버깅
- **목표:** 복합 데이터 타입의 기초를 배우고 데이터를 출력하는 법을 익힌다.
- **주요 내용:**
    - **Tuples:** 여러 타입의 값을 하나로 묶기, 패턴 매칭을 통한 분해(Destructuring)
    - **Arrays:** 동일한 타입의 고정 크기 집합
    - **Printing:**
        - `println!` 매크로 사용법 (`{}`, `{:?}`, `{:#?}`)
        - 디버그 트레이트(`Debug`)의 중요성
- **실습 지시:**
    - 튜플을 생성하고 각 요소를 변수에 할당하기
    - 배열의 인덱싱을 통해 값 접근하기
    - `println!("{:?}", ...)`를 사용하여 구조체나 복합 타입 출력 시도하기

### 16:10~18:00: 랩 (Lab) 및 과제
- **목표:** 배운 내용을 바탕으로 첫 코드를 작성한다.
- **활동:**
    - **첫 CLI 프로젝트 생성:** `cargo new calculator`
    - **개발 환경 체크리스트 완료:** `cargo check`, `cargo fmt` 실행 확인
    - **과제:** 사칙연산 계산기 뼈대 작성
        - 사용자로부터 숫자 두 개와 연산자를 입력받는 구조(매크로/함수 활용) 설계
        - `println!`을 사용하여 결과 출력하기
