# DXAX 실습 문제지

## 공통 제출 규칙

- 파일명은 `주차_일차_이름_산출물명` 형식으로 저장한다.
- 모든 실습 산출물에는 `문제`, `접근 방법`, `결과`, `다음 수정 포인트`를 포함한다.
- 실습 종료 10분 전에는 결과물 정리와 제출 형식 점검만 한다.

## 1주차 실습

### 1일차 업무 흐름 구조화

- 목표: 자신의 업무를 단계별 프로세스로 분해하고 DX 관점의 개선 후보를 찾는다.
- 준비물: 현재 담당 업무 목록, 자주 쓰는 문서 또는 메신저 기록
- 수행 단계:
  1. 하루 또는 일주일 단위로 반복되는 업무 3개를 적는다.
  2. 각 업무를 시작-중간-종료 단계로 세분화한다.
  3. 시간이 오래 걸리거나 반복이 많은 지점을 표시한다.
  4. 개선이 필요하다고 느끼는 이유를 한 줄씩 적는다.
- 기대 산출물: 개인 업무 프로세스 초안 1개, 개선 필요 지점 목록 3건 이상
- 시간 배분: 작성 25분, 정리 15분, 공유 10분
- 평가 기준:
  - 업무 단계가 실제 흐름 순서대로 정리돼 있는가
  - 병목 또는 반복 지점이 표시돼 있는가
  - 개선 필요 이유가 구체적으로 적혀 있는가

### 2일차 비효율 진단표 작성

- 목표: 비효율 업무를 시간, 빈도, 오류 가능성 기준으로 진단한다.
- 준비물: 1일차 업무 흐름도
- 수행 단계:
  1. 1일차 산출물에서 핵심 업무 1개를 고른다.
  2. 각 단계별 소요 시간과 반복 횟수를 적는다.
  3. 지연, 중복, 재작업이 생기는 이유를 분류한다.
  4. 개선 후보 업무를 우선순위 순으로 3개 정리한다.
- 기대 산출물: 비효율 진단표 1부, 우선 개선 후보 3건
- 시간 배분: 분석 20분, 우선순위 정리 20분, 공유 10분
- 평가 기준:
  - 진단 근거가 숫자나 사례로 제시되는가
  - 병목 원인이 구분돼 있는가
  - 개선 후보 우선순위가 논리적으로 설명되는가

### 3일차 AI 적용 우선순위 매트릭스

- 목표: AI 적용 효과와 실행 가능성을 함께 고려해 우선순위를 정한다.
- 준비물: 2일차 개선 후보 목록
- 수행 단계:
  1. 후보 업무별 기대 효과를 상중하로 평가한다.
  2. 데이터 준비도와 실행 난이도를 적는다.
  3. 위험 요소와 보안 이슈를 적는다.
  4. 파일럿 과제 1개와 장기 과제 1개를 선택한다.
- 기대 산출물: 우선순위 매트릭스, 파일럿 과제 선정표
- 시간 배분: 평가 20분, 비교 20분, 발표 10분
- 평가 기준:
  - 효과와 난이도를 함께 판단했는가
  - 위험 요소가 빠지지 않았는가
  - 파일럿 과제 선정 이유가 명확한가

### 4일차 문제 정의서 작성

- 목표: 해결할 문제를 현상, 원인, 기대 결과 기준으로 구조화한다.
- 준비물: 3일차 파일럿 과제 선정표
- 수행 단계:
  1. 현재 발생하는 문제 상황을 한 문단으로 적는다.
  2. 문제의 원인을 3가지 이하로 정리한다.
  3. 해결 후 기대하는 결과를 정량 또는 정성 지표로 적는다.
  4. 제약조건과 제외 범위를 적는다.
- 기대 산출물: 문제 정의서 1부
- 시간 배분: 작성 30분, 상호 검토 15분, 수정 5분
- 평가 기준:
  - 현상과 원인이 구분되는가
  - 기대 결과가 측정 가능한가
  - 제약조건이 명시되는가

### 5일차 요구사항 정의서 작성

- 목표: AI 적용에 필요한 입력 데이터와 출력 조건을 명확히 정의한다.
- 준비물: 4일차 문제 정의서
- 수행 단계:
  1. 필요한 입력 데이터 항목을 목록화한다.
  2. 사용할 수 없는 데이터와 보안 제약을 적는다.
  3. 원하는 출력 형식과 검수 기준을 적는다.
  4. 재사용 가능한 템플릿 구조로 정리한다.
- 기대 산출물: 요구사항 정의서 1부
- 시간 배분: 정리 25분, 검토 15분, 공유 10분
- 평가 기준:
  - 입력과 출력 요구사항이 모두 있는가
  - 제약조건이 현실적인가
  - 다음 주 프롬프트 설계로 바로 연결 가능한가

## 2주차 실습

### 6일차 기본 프롬프트 템플릿 설계

- 목표: 1주차 요구사항을 재현 가능한 프롬프트로 변환한다.
- 준비물: 문제 정의서, 요구사항 정의서
- 수행 단계:
  1. 역할, 목적, 입력값, 절차, 출력 형식을 나눈다.
  2. 필수 제약조건과 검수 기준을 추가한다.
  3. 동일 업무에 대해 초안용과 개선용 프롬프트를 분리한다.
  4. 동료와 교차 테스트해 이해 가능한지 확인한다.
- 기대 산출물: 프롬프트 템플릿 2종 이상
- 시간 배분: 설계 20분, 테스트 20분, 보완 10분
- 평가 기준:
  - 구조가 반복 사용 가능하게 정리됐는가
  - 검수 기준이 포함되는가
  - 타인이 읽어도 의미가 통하는가

### 7일차 업무 문서 초안과 개선본 작성

- 목표: AI로 문서 초안을 만들고 검수 기준으로 완성도를 높인다.
- 준비물: 6일차 프롬프트 템플릿
- 수행 단계:
  1. 이메일, 보고서, 제안서 중 1개 유형을 고른다.
  2. 초안 생성 프롬프트로 1차 문서를 만든다.
  3. 검수 체크리스트로 부족한 점을 표시한다.
  4. 개선 프롬프트로 최종본을 만든다.
- 기대 산출물: 초안 1부, 개선본 1부, 수정 메모
- 시간 배분: 초안 15분, 검수 15분, 개선 20분
- 평가 기준:
  - 초안과 최종본의 차이가 분명한가
  - 수정 이유가 설명되는가
  - 문서 구조가 목적에 맞는가

### 8일차 텍스트 요약과 인사이트 메모

- 목표: 긴 자료를 목적에 맞게 요약하고 실무 인사이트로 재구성한다.
- 준비물: 긴 기사, 보고서, 회의록 중 1개
- 수행 단계:
  1. 대상 독자와 요약 목적을 정한다.
  2. 요약 분량 기준을 정한다.
  3. AI로 요약본을 생성한다.
  4. 요약본을 바탕으로 인사이트 메모 3개를 작성한다.
- 기대 산출물: 요약본 1부, 인사이트 메모 3건
- 시간 배분: 기준 설정 10분, 요약 20분, 메모 작성 20분
- 평가 기준:
  - 요약이 목적에 맞게 압축됐는가
  - 핵심 내용 누락이 없는가
  - 인사이트가 후속 행동과 연결되는가

### 9일차 데이터 해석 메모 작성

- 목표: 샘플 데이터를 해석하고 비즈니스 언어로 설명한다.
- 준비물: 제공 샘플 데이터
- 수행 단계:
  1. 핵심 지표 3개를 선택한다.
  2. 변화나 차이를 읽고 가능한 해석을 적는다.
  3. 추가로 필요한 데이터를 적는다.
  4. 의사결정 메모 형식으로 정리한다.
- 기대 산출물: 데이터 해석 메모 1부
- 시간 배분: 지표 확인 15분, 해석 20분, 메모 정리 15분
- 평가 기준:
  - 지표와 해석이 구분돼 있는가
  - 과도한 단정 없이 가설로 설명하는가
  - 후속 질문이 포함되는가

### 10일차 Excel 데이터 요약 보고

- 목표: Excel 데이터 구조를 정리하고 AI로 요약 및 해석 보고를 작성한다.
- 준비물: Excel 샘플 파일
- 수행 단계:
  1. 열 이름, 결측값, 범주 값을 정리한다.
  2. 요약 프롬프트로 핵심 지표를 추출한다.
  3. 이상치 또는 주목 지점을 2개 이상 적는다.
  4. 해석 보고 초안을 작성한다.
- 기대 산출물: 데이터 요약 결과, 해석 보고 초안
- 시간 배분: 정제 20분, 요약 15분, 보고 작성 15분
- 평가 기준:
  - 데이터 구조 정리가 선행됐는가
  - 요약 결과가 핵심 지표 중심인가
  - 보고 초안이 조치 제안까지 포함하는가

## 3주차 실습

### 11일차 자동화 시나리오 초안

- 목표: 반복 업무를 자동화 흐름으로 재구성한다.
- 준비물: 2주차 산출물, 자동화 후보 업무
- 수행 단계:
  1. 자동화할 업무 1개를 고른다.
  2. Trigger와 Action을 순서대로 적는다.
  3. 사람이 최종 확인해야 하는 지점을 표시한다.
  4. 기대 효과를 시간, 품질, 속도 기준으로 적는다.
- 기대 산출물: 자동화 시나리오 초안 1부
- 시간 배분: 설계 25분, 검토 15분, 공유 10분
- 평가 기준:
  - 업무 흐름이 순서대로 보이는가
  - 사람 개입 지점이 구분되는가
  - 효과가 구체적으로 설명되는가

### 12일차 자동화 흐름도와 예외 처리 설계

- 목표: 자동화 시나리오를 예외 처리까지 포함한 흐름도로 구체화한다.
- 준비물: 11일차 시나리오 초안
- 수행 단계:
  1. 입력-처리-출력 흐름을 도식화한다.
  2. 오류 가능 지점을 3개 이상 표시한다.
  3. 예외 발생 시 재시도, 알림, 수동 전환 규칙을 적는다.
  4. 프롬프트 또는 규칙 문장을 흐름도에 연결한다.
- 기대 산출물: 자동화 흐름도 1부, 예외 처리 메모
- 시간 배분: 흐름도 25분, 예외 설계 20분, 리뷰 5분
- 평가 기준:
  - 정상 흐름과 예외 흐름이 모두 있는가
  - 입력과 출력이 명확한가
  - 실현 가능성이 검토됐는가

### 13일차 팀 협업 적용 시나리오

- 목표: 개인 자동화 흐름을 팀 협업 맥락으로 확장한다.
- 준비물: 12일차 흐름도
- 수행 단계:
  1. 팀에서 같이 쓰는 문서 또는 프로세스를 고른다.
  2. 역할별 입력자, 검토자, 승인자를 적는다.
  3. AI 지원이 필요한 협업 장면을 2개 이상 찾는다.
  4. 팀 적용 시 운영 규칙을 적는다.
- 기대 산출물: 팀 적용 시나리오, 협업 문서 초안
- 시간 배분: 설계 20분, 문서화 20분, 공유 10분
- 평가 기준:
  - 역할 분담이 명확한가
  - 개인 사용과 팀 사용의 차이가 반영되는가
  - 운영 규칙이 빠지지 않았는가

### 14일차 반복 작업 감소 전략

- 목표: 효율화 지표를 기준으로 반복 작업 감소 전략을 설계한다.
- 준비물: 13일차 협업 시나리오
- 수행 단계:
  1. 반복 작업 3개를 추린다.
  2. 각 작업의 현재 소요 시간과 품질 리스크를 적는다.
  3. 프롬프트, 템플릿, 자동화 중 적절한 방법을 매칭한다.
  4. 절감 효과와 운영 기준을 정리한다.
- 기대 산출물: 반복 작업 감소 전략표, 업무 정리 매뉴얼 초안
- 시간 배분: 분석 20분, 전략 수립 20분, 발표 10분
- 평가 기준:
  - 절감 대상이 명확한가
  - 해결 방법이 업무 특성과 맞는가
  - 품질 유지 장치가 포함되는가

### 15일차 변화 시나리오 설계

- 목표: AI 도입 이후 개인과 조직의 역할 변화를 구조화한다.
- 준비물: 14일차 전략표
- 수행 단계:
  1. 현재 역할과 미래 역할의 차이를 적는다.
  2. 예상 저항 요소를 3개 이상 적는다.
  3. 교육, 커뮤니케이션, 검수 체계 대응책을 적는다.
  4. 변화 시나리오를 발표용 한 장으로 요약한다.
- 기대 산출물: 변화 시나리오 1부, 역할 변화 정리표
- 시간 배분: 작성 25분, 요약 15분, 발표 10분
- 평가 기준:
  - 변화 전후 차이가 명확한가
  - 저항과 대응이 함께 설계됐는가
  - 실제 조직에 적용 가능한 언어로 쓰였는가

## 4주차 실습

### 16일차 업무용 이미지 프롬프트 설계

- 목표: 실무 목적에 맞는 이미지 프롬프트를 만들고 결과물을 생성한다.
- 준비물: 홍보물, 발표자료, 카드뉴스 중 1개 시나리오
- 수행 단계:
  1. 이미지 사용 목적과 대상 독자를 적는다.
  2. 스타일, 구도, 색감, 금지 요소를 정한다.
  3. 프롬프트를 작성하고 결과물을 생성한다.
  4. 업무 목적 기준으로 수정 포인트를 기록한다.
