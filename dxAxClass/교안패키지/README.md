# DXAX 교안 패키지

이 폴더는 이미지형 PDF만으로는 확인하기 어려웠던 강의 운영 정보를 텍스트 기반으로 보강한 교안 패키지다. 전체 구조는 `일일 교육계획안.md`의 20일 흐름을 그대로 따르며, QA 보고서에서 지적된 발표자 스크립트, 진행자 노트, 실습 문제지를 분리해 정리했다.

## 구성

- `발표자_스크립트.md`: 일차별 오프닝, 핵심 설명 포인트, 전환 멘트, 마무리 멘트
- `진행자_노트.md`: 일차별 준비물, 시간 운영 포인트, 질문 유도, 리스크 대응, 산출물 점검 기준
- `실습_문제지.md`: 일차별 실습 목표, 준비물, 단계, 기대 산출물, 시간 배분, 평가 기준
- `슬라이드_비주얼_앵커.md`: 렌더링된 PDF 슬라이드 첫 장을 기준으로 주차별 메시지와 교안 패키지의 연결 포인트를 정리한 검토 메모
- `../강의운영_스크립트/`: 20일 x 8교시 기준의 시간별 강의 스크립트, 참조 포인트, 수업 팁을 정리한 별도 운영 런북

## 운영 원칙

- 1주차는 업무 진단과 문제 정의를 위한 공통 언어를 만든다.
- 2주차는 1주차 산출물을 입력값으로 사용해 실무 결과물을 생산한다.
- 3주차는 2주차 결과물을 자동화와 협업 흐름으로 재구성한다.
- 4주차는 멀티모달과 n8n 구현을 거쳐 최종 실행 로드맵으로 회수한다.
- 모든 일차는 `이해 -> 적용 -> 리뷰 -> 다음 날 입력값 정리`의 리듬을 유지한다.

## 권장 사용 순서

1. 수업 전 `진행자_노트.md`로 환경, 시간, 위험 요소를 확인한다.
2. 수업 직전 또는 교시 전환 시 `../강의운영_스크립트/`의 해당 주차 문서로 시간별 멘트와 연결 포인트를 확인한다.
3. 수업 중 `발표자_스크립트.md`로 일차 전체 메시지와 오프닝, 마무리 톤을 맞춘다.
4. 실습 직전 `실습_문제지.md`를 배포하거나 화면 공유해 제출 기준을 명확히 한다.
5. 8교시에는 반드시 당일 산출물을 다음 날 입력값으로 이름 붙여 저장한다.
