# Day 10. Generics와 Traits

## 1. 강의 개요
- **일자:** Day 10
- **목표:** Rust의 강력한 추상화 도구인 Generics와 Traits를 배우고, 코드 재사용성과 유연성을 극대화하는 방법을 익힌다.

---

## 2. 시간대별 세션

### 09:00~10:30: Generics - 코드의 재사용성 높이기
- **목표:** 동일한 로직을 다양한 타입에 대해 적용하는 방법을 배운다.
- **주요 내용:**
    - **Generic Types:** 함수와 구조체에서 `<T>`를 사용하는 법.
    - **Generic Functions:** 다양한 타입을 인자로 받는 범용 함수 작성.
    - **Generic Structs & Enums:** 다양한 데이터를 담을 수 있는 범용 데이터 구조 정의.
- **실습 지시:**
    - `Stack<T>` 구조체를 만들어 다양한 타입을 담을 수 있는지 확인하기.

### 10:45~12:00: Traits - 동작의 추상화
- **목표:** 특정 타입이 수행할 수 있는 '행위'를 정의하는 법을 배운다.
- **주요 내용:**
    - **Trait Definition:** `trait` 키워드를 사용하여 공통 인터페이스 정의.
    - **Trait Implementation:** 특정 타입에 대해 `impl Trait for Type` 방식으로 동작 구현.
    - **Default Implementations:** 트레이트 내에 기본 동작 정의하기.
- **실습 지시:**
    - `Summary` 트레이트를 만들고, `Article`과 `Tweet` 구조체에 각각 구현해보기.

### 13:00~14:30: Trait Bounds - 제약 조건이 있는 Generics
- **목표:** Generic 타입이 특정 동작을 할 수 있음을 보장하는 방법을 배운다.
- **주요 내용:**
    - **Trait Bounds:** `fn foo<T: Display>(item: T)`와 같이 타입에 제약 걸기.
    - **Multiple Bounds:** `T: Display + Clone`과 같이 여러 제약 결합하기.
    - **`where` Clause:** 복잡한 제약 조건을 깔끔하게 정리하기.
- **실습 지시:**
    - 특정 트레이트를 구현한 타입만 인자로 받을 수 있는 범용 함수 작성하기.

### 14:45~16:00: 고급 트레이트 패턴 (Derive, Orphan Rule)
- **목표:** 트레이트를 더 효율적으로 활용하고 Rust의 규칙을 이해한다.
- **주요 내용:**
    - **`#[derive(...)]`:** `Debug`, `Clone`, `PartialEq` 등 자주 쓰는 트레이트 자동 구현.
    - **The Orphan Rule:** 외부 크레이트의 타입에 외부 크레이트의 트레이트를 구현할 수 없는 규칙 이해.
    - **Trait Objects (`dyn Trait`):** 컴파일 타임이 아닌 런타임에 다형성(Polymorphism)을 구현하는 법.
- **실습 지시:**
    - `derive`를 사용하여 구조체에 `Debug` 기능 추가하기.
    - `Box<dyn Summary>`를 사용하여 서로 다른 타입을 하나의 벡터에 담기.

### 16:10~18:00: 랩 (Lab) 및 과제
- **목표:** Generics와 Traits를 결합하여 확장 가능한 시스템을 설계한다.
- **활동:**
    - **범용 데이터 처리기 구현:**
        - `Printable` 트레이트를 정의하고, 이를 구현한 다양한 타입(숫자, 문자열, 사용자 정의 구조체)을 하나의 함수에서 처리하는 프로그램 작성.
    - **과제: 확장 가능한 플러그인 시스템 설계**
        - `Plugin` 트레이트를 정의하고, 다양한 기능을 수행하는 플러그인들을 등록하고 실행할 수 있는 구조 설계.
