# Day 7. 소유권(Ownership)과 빌림(Borrowing) 심화

## 1. 강의 개요
- **일자:** Day 7
- **목표:** 소유권과 빌림의 핵심 규칙을 심화 학습하고, 복잡한 데이터 구조에서 발생하는 참조 문제를 해결하는 능력을 기른다.

---

## 2. 시간대별 세션

### 09:00~10:30: 깊은 복사(Deep Copy) vs 얕은 복사(Shallow Copy)
- **목표:** 메모리 상에서 데이터가 실제로 어떻게 복제되는지 이해한다.
- **주요 내용:**
    - **Shallow Copy:** 포인터(주소값)만 복사되어 동일한 힙 데이터를 가리키는 위험성.
    - **Deep Copy (`Clone`):** 새로운 힙 메모리를 할당하고 데이터를 모두 복제하는 과정.
    - Rust에서 `Clone` 트레이트의 역할과 성능 비용 고려하기.
- **실습 지시:**
    - `Vec<String>`을 `clone()` 했을 때와 단순히 대입했을 때의 차이 확인하기.

### 10:45~12:00: 참조의 수명(Lifetimes) 기초
- **목표:** 참조가 유효한 범위를 컴파일러에게 알려주는 '수명'의 개념을 맛본다.
- **주요 내용:**
    - **Lifetime의 필요성:** dangling reference(허공을 가리키는 참조) 방지.
    - **Lifetime Annotation (`'a`):** 함수 시그니처에서 참조 간의 관계 명시하기.
    - 컴파일러가 추론할 수 있는 범위와 명시적 선언이 필요한 경우.
- **실습 지시:**
    - 두 개의 문자열 슬라이스 중 더 긴 것을 반환하는 함수를 작성하고, 발생하는 Lifetime 에러 해결해보기.

### 13:00~14:30: 스마트 포인터 (Smart Pointers) I: `Box<T>`
- **목표:** 힙 메모리에 데이터를 할당하고 관리하는 가장 기본적인 스마트 포인터를 배운다.
- **주요 내용:**
    - **`Box<T>`:** 데이터를 힙에 할당하고 스택에는 포인터만 남기는 법.
    - **Recursive Types:** 재귀적 구조(예: Linked List)를 정의할 때 `Box`가 필수적인 이유.
- **실습 지시:**
    - `Box`를 사용하여 간단한 재귀적 트리 구조(Node) 정의해보기.

### 14:45~16:00: 스마트 포인터 II: `Rc<T>` (Reference Counting)
- **목표:** 여러 소유자가 하나의 데이터를 공유해야 하는 상황을 해결한다.
- **주요 내용:**
    - **`Rc<T>` (Reference Counted):** 힙에 있는 데이터를 여러 곳에서 참조할 수 있게 하는 참조 카운팅 메커니즘.
    - 참조 카운트의 증가와 감소 원리.
    - `Rc<T>`의 한계: 불변성(Immutability)만 보장한다는 점.
- **실습 지시:**
    - 여러 개의 변수가 동일한 `Rc<String>`을 가리키도록 설정하고, 카운트 확인하기.

### 16:10~18:00: 랩 (Lab) 및 과제
- **목표:** 소유권과 스마트 포인터를 조합하여 복잡한 데이터 구조를 설계한다.
- **활동:**
    - **공유 데이터 그래프(Graph) 기초 구현:**
        - 노드(Node)들이 서로를 참조할 수 있는 간단한 구조 설계.
        - `Rc<T>`를 사용하여 여러 노드가 하나의 노드를 가리키도록 구현.
    - **과제: `Box`를 이용한 재귀적 리스트 구현**
        - `enum List { Cons(i32, Box<List>), Nil }` 형태의 구조를 구현하고, 리스트에 요소를 추가하고 순회하는 기능 작성.
