# Day 7. Lifetime Basics & Sharing (RC/Box)

## 1. 강의 개요
- **일자:** Day 7
- **목표:** Rust의 메모리 관리의 심화 개념인 Lifetimes, Box, Rc를 배우고, 데이터의 수명과 공유 방식을 이해한다.

---

## 2. 시간대별 세션

### 09:00~10:30: Lifetimes - 참조의 유효성 보장
- **목표:** 컴파일러가 참조의 유효성을 판단하는 기준을 이해하고, 명시적인 Lifetime Annotation을 사용하는 법을 배운다.
- **주요 내용:**
    - **Lifetime의 개념:** 참조가 가리키는 데이터가 참조보다 오래 살아남아야 한다는 원칙.
    - **Lifetime Elision:** 컴파일러가 자동으로 Lifetime을 추론하는 규칙.
    - **Lifetime Annotation (`'a`):** 함수 시그니처에서 명시적으로 수명을 표현하는 법.
- **실습 지시:**
    - 두 문자열 중 긴 것을 반환하는 함수를 작성하고, Lifetime Annotation을 사용하여 컴파일 에러 해결하기.

### 10:45~12:00: Box<T> - Heap 할당과 재귀적 구조
- **목표:** 데이터를 스택이 아닌 힙에 할당하고 관리하는 법을 배운다.
- **주요 내용:**
    - **Box<T>의 역할:** 힙 메모리 할당, 크기가 컴파일 타임에 결정되지 않는 타입(Recursive types) 처리.
    - **재귀적 데이터 구조:** `Box`를 사용하여 Linked List나 Tree 구조 정의하기.
- **실습 지시:**
    - `Box`를 사용하여 간단한 단항 연산 트리(Unary Tree) 또는 재귀적 리스트 구현하기.

### 13:00~14:30: Rc<T> - Reference Counting (공유 소유권)
- **목표:** 단일 소유자 원칙을 넘어, 여러 곳에서 데이터를 공유하는 법을 배운다.
- **주요 내용:**
    - **`Rc<T>` (Reference Counted):** 힙에 저장된 데이터를 여러 소유자가 가질 수 있게 하는 스마트 포인터.
    - **Reference Counting 원리:** `clone()` 시 카운트 증가, `drop` 시 카운트 감소.
    - **불변성(Immutability):** `Rc<T>` 내부의 데이터는 기본적으로 불변임을 이해하기.
- **실습 지시:**
    - `Rc`를 사용하여 여러 노드가 하나의 데이터를 가리키는 그래프 구조의 기초 만들기.

### 14:45~16:00: Advanced Sharing (Rc + RefCell)
- **목표:** 공유되는 데이터에 대해 가변성(Mutability)을 확보하는 방법을 배운다.
- **주요 내용:**
    - **Interior Mutability 패턴:** 외부 인터페이스는 불변이지만 내부 데이터는 변경 가능한 구조.
    - **`RefCell<T>`:** 런타임에 빌림 규칙을 검사하는 스마트 포인터.
    - **`Rc<RefCell<T>>` 조합:** 공유 가능하면서도 수정 가능한 데이터 구조 만들기.
- **실습 지시:**
    - `Rc<RefCell<T>>`를 사용하여 여러 곳에서 공유되고 수정될 수 있는 공유 카운터 또는 공유 리스트 구현하기.

### 16:10~18:00: 랩 (Lab) 및 과제
- **목표:** Lifetime과 스마트 포인터를 결합하여 복잡한 데이터 구조를 설계한다.
- **활동:**
    - **공유 그래프 구조 구현:**
        - `Rc<RefCell<Node>>`를 사용하여 노드 간에 양방향 참조가 가능한 그래프를 구현하고, 특정 노드를 찾아 데이터를 수정하는 프로그램 작성.
    - **과제: 복합 데이터 구조 설계**
        - `Box`를 통한 재귀적 구조와 `Rc`를 통한 데이터 공유가 혼합된 복잡한 트리/그래프 구조를 설계하고, Lifetime Annotation이 필요한 지점을 찾아 구현하기.
