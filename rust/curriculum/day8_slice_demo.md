# Day 8. Slices and String Parsing

## 1. 강의 개요
- **일자:** Day 8
- **목표:** 슬라이스(Slices)의 개념을 완벽히 이해하고, 문자열 및 배열 데이터를 효율적으로 파싱하고 처리하는 기술을 습득한다.

---

## 2. 시간대별 세션

### 09:00~10:30: 슬라이스(Slices) 기초
- **목표:** 데이터의 소유권을 갖지 않고 일부를 참조하는 슬라이스의 원리를 이해한다.
- **주요 내용:**
    - **Slice Type (`&[T]`):** 연속된 메모리 영역을 가리키는 참조의 구조.
    - **String Slices (`&str`):** UTF-8 문자열 데이터의 일부를 가리키는 방식.
    - **Memory Layout:** 포인터와 길이(length)로 구성된 슬라이스의 내부 구조.
- **실습 지시:**
    - 배열과 문자열에서 슬라이스를 생성하고, 슬라이스를 통해 데이터에 접근하기.

### 10:45~12:00: 문자열 파싱 (String Parsing)
- **목표:** 문자열을 분리하고 원하는 데이터로 변환하는 방법을 배운다.
- **주요 내용:**
    - **Split Methods:** `split`, `split_whitespace`, `split_terminator` 등의 활용.
    - **Iterator 기반 파싱:** 파싱된 결과를 이터레이터로 처리하여 메모리 효율 높이기.
    - **Parse Trait:** 문자열을 숫자나 다른 타입으로 변환하기 (`str::parse`).
- **실습 지시:**
    - 공백으로 구분된 숫자 문자열을 읽어 `Vec<i32>`로 변환하는 함수 작성하기.

### 13:00~14:30: 고급 파싱 패턴과 에러 처리
- **목표:** 복잡한 형식의 데이터를 안전하게 파싱하는 방법을 익힌다.
- **주요 내용:**
    - **Pattern Matching with Iterators:** `next()`, `collect()`, `filter()`를 조합한 파싱.
    - **Handling Parse Errors:** `parse()` 호출 시 발생하는 에러를 `Result`로 처리하기.
- **실습 지시:**
    - `key:value` 형태의 문자열 라인을 파싱하여 `HashMap`으로 만드는 프로그램 작성하기.

### 14:45~16:00: 실전: 데이터 추출 및 변환
- **목표:** 실제 데이터 세트에서 필요한 정보를 추출하는 실습을 수행한다.
- **주요 내용:**
    - **Pattern Matching & Regex (선택적):** 정규표현식을 이용한 정교한 파싱 맛보기.
    - **Zero-copy Parsing:** 원본 데이터의 소유권을 유지하면서 슬라이스만 사용하여 성능 최적화하기.
- **실습 지시:**
    - CSV 형식의 문자열 데이터를 파싱하여 구조체 리스트로 변환하기.

### 16:10~18:00: 랩 (Lab) 및 과제
- **목표:** 슬라이스와 파싱 기술을 결합하여 완성도 있는 툴을 만든다.
- **활동:**
    - **간이 로그 파서(Log Parser) 구현:**
        - `[TIMESTAMP] LEVEL: MESSAGE` 형식의 로그 문자열을 입력받아, 각 필드를 슬라이스로 분리하고 분석하는 프로그램 작성.
    - **과제: 커스텀 설정 파일 파서 설계**
        - 사용자 정의 문법(예: 주석 `#` 지원, 공백 허용)을 가진 설정 파일을 읽어 `HashMap<String, String>`으로 파싱하는 견고한 파서 완성하기.
