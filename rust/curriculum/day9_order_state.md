# Day 9. Library System & State Management

## 1. 강의 개요
- **일자:** Day 9
- **목표:** Rust의 모듈 시스템을 이해하고, 구조체와 트레이트를 활용하여 실제적인 라이브러리 형태의 도메인 로직을 설계한다.

---

## 2. 시간대별 세션

### 09:00~10:30: 모듈 시스템 (Module System)
- **목표:** 코드를 논리적 단위로 나누고 관리하는 법을 배운다.
- **주요 내용:**
    - **`mod`, `use`, `pub` 키워드:** 모듈 선언, 사용, 공개 범위 설정.
    - **File-based Modules:** 파일 시스템을 활용한 모듈 구조화.
    - **Crate Structure:** 라이브러리 크레이트와 바이너리 크레이트의 차이.
- **실습 지시:**
    - `book.rs`, `member.rs` 파일을 만들고 이를 `lib.rs`에서 모듈로 구성하기.

### 10:45~12:00: 도메인 모델링 (Domain Modeling)
- **목표:** 도메인 요구사항을 Rust의 타입 시스템으로 변환한다.
- **주요 내용:**
    - **Structs & Enums:** 도메인 객체(Book, Member, Status) 정의.
    - **Implementation Blocks:** 데이터와 행위의 결합 (`impl`).
    - **Encapsulation:** 필드의 가시성(`pub`)을 조절하여 데이터 무결성 유지하기.
- **실습 지시:**
    - 도서관 시스템의 핵심 객체(Book, Member)를 정의하고 기본적인 메서드 구현하기.

### 13:00~14:30: 라이브러리 인터페이스 설계
- **목표:** 사용자가 사용하기 편리하고 안전한 API를 설계한다.
- **주요 내용:**
    - **Constructor Pattern:** `new()` 메서드를 통한 객체 생성 관리.
    - **Result-based API:** 실패 가능성이 있는 동작(대여, 반납)에 `Result` 적용.
    - **Trait-based Abstraction:** 검색 인터페이스 등을 트레이트로 추상화하기.
- **실습 지시:**
    - 도서관 관리자(LibraryManager) 구조체를 만들고 도서 검색 및 대여 기능을 API로 제공하기.

### 14:45~16:00: 상태 관리와 데이터 무결성
- **목표:** 시스템의 상태 변화를 안전하게 관리한다.
- **주요 내용:**
    - **Ownership & State Transitions:** 객체의 상태가 변할 때 소유권이나 참조를 어떻게 다룰 것인가.
    - **Error Handling in Library:** 대여 가능 여부, 회원 상태 등을 에러로 처리하기.
- **실습 지시:**
    - 이미 대여 중인 책을 다시 대여하려고 할 때 에러를 반환하는 로직 구현하기.

### 16:10~18:00: 랩 (Lab) 및 과제
- **목표:** 모듈화된 라이브러리 형태의 도서 관리 시스템을 완성한다.
- **활동:**
    - **도서 관리 라이브러리 완성:**
        - `library_system`이라는 이름의 크레이트를 완성하고, `main.rs`에서 이 라이브러리를 사용하여 도서 대여/반납 시나리오를 테스트하기.
    - **과제: 확장 가능한 도서관 시스템 설계**
        - 디지털 도서(e-book)와 물리 도서(physical book)를 구분하여 관리할 수 있도록 트레이트와 열거형을 활용하여 구조 확장하기.
