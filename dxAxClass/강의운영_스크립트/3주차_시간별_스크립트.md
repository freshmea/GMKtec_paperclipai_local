# 3주차 시간별 강의운영 스크립트

## 11일차

- 일일 목표: 자동화 후보 업무를 선별하고 시나리오 초안을 만든다.
- 종료 산출물: 자동화 후보 업무 목록, 자동화 시나리오 초안

### 1교시 3주차 오리엔테이션
- 스크립트: "3주차는 개인 생산성을 조직 실행으로 연결하는 주간입니다. 지난주 결과물 중 자동화 가치가 있는 흐름을 다시 보겠습니다."
- 참조할 내용: 2주차 산출물, 3주차 목표, 자동화와 협업 흐름
- 수업 팁: 자동화는 기술 수업이 아니라 업무 설계 수업이라는 점을 먼저 못 박는다.

### 2교시 노코드 자동화 개념 이해
- 스크립트: "노코드 자동화는 개발 대체가 아니라 연결 비용을 낮추는 방식입니다. 자동화의 철학부터 간단히 정리하겠습니다."
- 참조할 내용: 노코드 자동화 개념, 장단점, 활용 장면
- 수업 팁: 화려한 데모보다 실제 반복 업무 감소 사례를 먼저 제시한다.

### 3교시 자동화 가능 업무 식별
- 스크립트: "규칙 기반 업무부터 자동화 후보가 됩니다. 예외가 많고 판단이 복잡한 일은 보류하는 것이 맞습니다."
- 참조할 내용: 규칙 기반 업무, 반복 빈도, 입력 안정성
- 수업 팁: 모든 걸 자동화하려는 분위기가 생기면 사람 판단 구간을 다시 표시하게 한다.

### 4교시 반복 업무 자동화 사례 분석
- 스크립트: "단순 반복 업무의 자동화는 가장 빠른 성과를 만듭니다. 작은 사례를 구조로 읽어 보겠습니다."
- 참조할 내용: 반복 업무 사례, 입력-처리-출력 흐름
- 수업 팁: 사례를 보여 줄 때 반드시 시작 조건과 종료 조건을 짚는다.

### 5교시 비효율 업무 자동화 사례 분석
- 스크립트: "병목 해결형 자동화는 단순 반복보다 설계가 어렵지만 효과가 큽니다. 어떤 조건에서 가능한지 보겠습니다."
- 참조할 내용: 병목 해결 사례, 승인 지연, 정보 탐색 감소
- 수업 팁: 사례의 핵심을 `없앤 단계`와 `새로 생긴 검수 단계`로 요약해 준다.

### 6교시 업무 자동화 시나리오 이해
- 스크립트: "자동화는 Trigger와 Action을 기준으로 생각하면 단순해집니다. 무엇이 시작 신호이고 어떤 결과가 나와야 하는지 정하겠습니다."
- 참조할 내용: Trigger, Action, 조건 분기, 검수 지점
- 수업 팁: 기술 용어 설명보다 일상 업무 예시로 먼저 감을 잡게 한다.

### 7교시 [실습] 자동화 시나리오 초안
- 스크립트: "각자 업무 흐름을 기준으로 자동화 시나리오를 그려 보겠습니다. 완전 자동화보다 안전한 반자동부터 시작해도 됩니다."
- 참조할 내용: 시나리오 카드, 자동화 후보 목록, 흐름도 템플릿
- 수업 팁: 막히는 사람은 2주차 산출물 생성 흐름을 예제로 전환하게 한다.

### 8교시 11일차 종합 정리
- 스크립트: "내일은 오늘 시나리오를 더 구체적인 구조와 예외 처리 규칙으로 바꿉니다. 입력값과 검수 지점을 빠뜨리지 말고 저장해 주세요."
- 참조할 내용: 시나리오 초안, 입력값, 검수 기준
- 수업 팁: 보안과 오류 관리 포인트를 마지막에 다시 상기시킨다.

## 12일차

- 일일 목표: 자동화 흐름도와 예외 처리 규칙을 포함한 설계안을 완성한다.
- 종료 산출물: 자동화 흐름도, 프롬프트 포함 설계안

### 1교시 자동화 설계의 논리 구조
- 스크립트: "자동화가 실제로 작동하려면 정상 흐름뿐 아니라 판단 로직이 구조화되어 있어야 합니다."
- 참조할 내용: 입력-처리-출력 구조, 검수, 승인, 예외 처리
- 수업 팁: 설계는 그림이 아니라 설명 가능한 논리라는 점을 계속 강조한다.

### 2교시 AI와 외부 도구 연계 원리
- 스크립트: "API와 Webhook은 복잡한 기술이 아니라 연결 방식의 차이입니다. 어떤 상황에서 어떤 연결이 필요한지 이해하면 충분합니다."
- 참조할 내용: API, Webhook, 시스템 간 데이터 이동
- 수업 팁: 네트워크 설명보다 `누가 먼저 신호를 보내는가`로 설명하면 이해가 빠르다.

### 3교시 데이터 파이프라인 시각화
- 스크립트: "자동화 흐름은 입력값이 어디서 오고 어디로 나가는지 보일 때 설계가 안정됩니다."
- 참조할 내용: 데이터 파이프라인, 입력원, 저장 위치, 결과물 전달 방식
- 수업 팁: 화살표마다 데이터 이름을 적게 하면 설계 품질이 올라간다.

### 4교시 [실습] 자동화 흐름 설계 1
- 스크립트: "시나리오별 워크플로우 초안을 그리겠습니다. 먼저 정상 경로만 짧게 그린 뒤 세부 요소를 추가합시다."
- 참조할 내용: 워크플로우 템플릿, 기본 노드 개념
- 수업 팁: 처음부터 복잡하게 그리지 말고 최소 성공 흐름부터 작성하게 한다.

### 5교시 [실습] 자동화 흐름 설계 2
- 스크립트: "이제 AI 프롬프트와 조건부 로직을 붙이겠습니다. 사람 검수 지점과 자동 처리 지점을 구분해 주세요."
- 참조할 내용: 프롬프트 삽입 위치, 조건부 분기, 승인 단계
- 수업 팁: AI가 생성하는 출력은 항상 검수 주체를 붙이게 한다.

### 6교시 예외 상황 및 오류 제어
- 스크립트: "자동화 설계의 완성도는 실패했을 때 드러납니다. 입력값이 없거나 형식이 틀릴 때 무엇이 일어나야 하는지 정리하겠습니다."
- 참조할 내용: 예외 처리, 필터링, 재시도, 알림
- 수업 팁: 정상 경로보다 예외 경로 발표를 더 많이 시키면 실무성이 높아진다.

### 7교시 업무 자동화 구조 정리
- 스크립트: "설계안을 문서 형태로 정리하겠습니다. 나중에 다른 사람이 읽어도 같은 흐름을 이해할 수 있어야 합니다."
- 참조할 내용: 표준 설계 문서 구조, 입력/출력/예외/책임자 항목
- 수업 팁: 그림과 문장을 같이 남기게 해야 전달력이 생긴다.

### 8교시 설계안 리뷰 및 피드백
- 스크립트: "내일은 개인 자동화를 팀 협업 흐름으로 확장합니다. 오늘 설계안에서 팀과 연결될 지점을 표시해 두세요."
- 참조할 내용: 설계안 리뷰 포인트, 협업 확장 후보
- 수업 팁: 피드백은 기술 난이도보다 실현 가능성과 운영 부담 중심으로 준다.

## 13일차

- 일일 목표: AI 활용을 개인 생산성에서 팀 협업 규칙으로 확장한다.
- 종료 산출물: 협업 문서 초안, 팀 적용 시나리오

### 1교시 협업 환경과 AI의 역할
- 스크립트: "AI는 개인 도구일 때보다 팀 규칙 안에 들어갈 때 더 큰 효과를 냅니다. 오늘은 팀 생산성 관점에서 다시 보겠습니다."
- 참조할 내용: 협업 환경, 팀 생산성, 공통 규칙의 필요성
- 수업 팁: 개인 편의보다 공유 가능성과 표준화 관점으로 전환시킨다.

### 2교시 협업 도구와 AI 결합 개념
- 스크립트: "협업 도구에 AI를 붙인다는 것은 기능 추가가 아니라 흐름 단순화입니다. 어떤 도구가 어떤 장면에서 유효한지 보겠습니다."
- 참조할 내용: 메신저, 문서, 일정, 프로젝트 도구와 AI 결합 구조
- 수업 팁: 특정 제품 설명으로 새지 않게 하고 역할 수준에서만 다룬다.

### 3교시 협업 상황별 AI 활용 사례
- 스크립트: "회의, 일정, 프로젝트 관리처럼 팀이 함께 겪는 장면에서 AI가 무엇을 대신할 수 있는지 사례로 보겠습니다."
- 참조할 내용: 협업 사례, 역할 분담, 검수와 승인
- 수업 팁: 사례마다 `누가 입력하고 누가 검토하는가`를 꼭 짚는다.

### 4교시 [실습] 협업 업무 지원 1
- 스크립트: "팀 아젠다와 협업 문서 초안을 만들어 보겠습니다. 개인 문서를 팀 문서로 바꾸려면 정보 구조가 달라져야 합니다."
- 참조할 내용: 팀 문서 양식, 아젠다 템플릿
- 수업 팁: 공유 위치와 버전 관리 방식을 같이 적게 한다.

### 5교시 팀 커뮤니케이션 최적화
- 스크립트: "메신저와 이메일은 속도보다 정렬 비용이 더 중요합니다. 소통을 줄이는 것이 아니라 다시 묻는 비용을 줄이는 방향으로 보겠습니다."
- 참조할 내용: 커뮤니케이션 템플릿, 메시지 구조화
- 수업 팁: 좋은 메시지 예시를 보여 주고 어떤 재질문을 줄였는지 분석하게 한다.

### 6교시 공동 지식 베이스 구축
- 스크립트: "팀의 지식은 쌓이는 것보다 찾을 수 있어야 의미가 있습니다. 저장과 검색 구조를 함께 생각합시다."
- 참조할 내용: 지식 베이스 구조, 태그, 문서 표준화
- 수업 팁: 도구보다 문서명 규칙과 분류 기준부터 정하게 한다.

### 7교시 [실습] 협업 업무 지원 2
- 스크립트: "협업 상황에서 AI 어시스턴트 활용을 시뮬레이션하겠습니다. 역할, 입력자, 검토자, 공유 위치를 모두 표시해 주세요."
- 참조할 내용: 협업 시나리오 카드, 팀 적용 체크리스트
- 수업 팁: 팀 사례가 없으면 회의 준비-회의록-후속 조치 흐름으로 공통 과제를 준다.

### 8교시 13일차 종합 정리 및 공유
- 스크립트: "내일은 팀 차원의 효율화 체계를 정리합니다. 오늘 문서에서 표준화할 수 있는 부분을 표시해 두세요."
- 참조할 내용: 팀 적용 시나리오, 표준화 후보
- 수업 팁: 공유 시간에는 멋진 사례보다 운영 조건이 명확한 사례를 먼저 다룬다.

## 14일차

- 일일 목표: 반복 작업 감소 전략과 업무 정리 체계를 만든다.
- 종료 산출물: 반복 업무 감소 전략, 업무 정리 매뉴얼 초안

### 1교시 문서·업무 효율화의 핵심 원리
- 스크립트: "효율화는 더 빨리 하는 것이 아니라 덜 흔들리게 하는 것입니다. 오늘은 절감 기준과 안정화 기준을 함께 보겠습니다."
- 참조할 내용: 효율화 지표, 시간 절감, 재작업 감소, 품질 안정
- 수업 팁: 속도만 강조하는 분위기를 잡고 오류 감소도 지표로 넣는다.

### 2교시 문서·업무 효율화 방법론
- 스크립트: "프롬프트 템플릿과 업무 최적화 체계는 따로 놀지 않습니다. 어떤 문서를 어떻게 반복 사용할지 기준을 세우겠습니다."
- 참조할 내용: 템플릿화, 표준화, 체크리스트화
- 수업 팁: 기존 산출물 중 반복 가능한 부분을 직접 표시하게 한다.

### 3교시 AI 활용 반복 작업 식별
- 스크립트: "지금까지 만든 결과물 중 대체 가능한 반복 작업을 다시 찾겠습니다. 이미 자동화 후보가 아닌지 함께 보세요."
- 참조할 내용: 반복 작업 분류, 대체 가능성, 위험성
- 수업 팁: 자동화와 수동 표준화 중 어느 쪽이 맞는지 비교하게 한다.

### 4교시 [실습] 반복 작업 감소 전략 수립
- 스크립트: "시간 절감 시나리오를 설계하겠습니다. 줄일 단계와 남길 단계, 검수 기준을 함께 적어 주세요."
- 참조할 내용: 절감 전략 양식, 전후 비교표
- 수업 팁: 목표 시간을 과장하지 않게 현재 기준 시간을 먼저 쓰게 한다.

### 5교시 AI 기반 고품질 문서 고도화
- 스크립트: "초안 이후의 고도화가 업무 품질을 결정합니다. 결과를 더 좋게 만드는 반복 패턴을 정리하겠습니다."
- 참조할 내용: 수정 루프, 품질 체크 항목, 리뷰 패턴
